## Authentication

All API requests require authentication via an API key passed in the `Authorization` header:

```javascript
const response = await fetch('https://api.nebula.ai/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.NEBULA_API_KEY}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    model: 'nebula-3-pro',
    messages: [{ role: 'user', content: 'Hello!' }],
  }),
});
```

Or using the SDK:

```javascript
import { NebulaClient } from '@nebula-api/sdk';

const client = new NebulaClient({
  apiKey: process.env.NEBULA_API_KEY,
});
```

## Chat Completions

The most common endpoint. Send a conversation history and receive the model's response.

```javascript
const response = await client.chat.completions.create({
  model: 'nebula-3-pro',
  messages: [
    { role: 'system', content: 'You are a helpful assistant.' },
    { role: 'user', content: 'Explain quantum computing in simple terms.' },
  ],
  temperature: 0.7,
  max_tokens: 1000,
});

console.log(response.choices[0].message.content);
```

### Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `model` | string | — | The model to use |
| `messages` | array | — | Conversation history |
| `temperature` | number | 0.7 | Randomness (0-2) |
| `max_tokens` | number | 4096 | Maximum response length |
| `top_p` | number | 1.0 | Nucleus sampling |
| `stream` | boolean | false | Enable streaming |

## Streaming Responses

For real-time output, enable streaming:

```javascript
const stream = await client.chat.completions.create({
  model: 'nebula-3-pro',
  messages: [{ role: 'user', content: 'Write a short story.' }],
  stream: true,
});

for await (const chunk of stream) {
  const content = chunk.choices[0]?.delta?.content || '';
  process.stdout.write(content);
}
```

## Error Handling

```javascript
import { NebulaClient, NebulaError } from '@nebula-api/sdk';

try {
  const response = await client.chat.completions.create({
    model: 'nebula-3-pro',
    messages: [{ role: 'user', content: 'Hello!' }],
  });
} catch (error) {
  if (error instanceof NebulaError) {
    console.error(`API Error: ${error.status} - ${error.message}`);

    if (error.status === 429) {
      console.log('Rate limited. Retrying after delay...');
    }
  }
}
```

<Callout type="info">
  The SDK automatically retries on transient errors (5xx) with exponential backoff. You only need to handle client errors (4xx).
</Callout>

## Rate Limits

| Plan | Requests/min | Tokens/min |
|------|-------------|------------|
| Free | 20 | 40,000 |
| Pro | 200 | 400,000 |
| Enterprise | Custom | Custom |

When rate limited, the API returns a `429` status with a `Retry-After` header.
