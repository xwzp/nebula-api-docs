本指南介绍如何通过 "Cloudflare for SaaS + 优选 IP 服务" 提升 Hapi 的访问速度，绕过拥堵节点。

## 解决的问题

默认 Tunnel 配置可能存在以下问题：

- Anycast IP 在国内可能被路由到美国或欧洲
- ISP QoS 限流降低访问速度
- 延迟可能达到数百毫秒

优选 IP 方案可以将延迟从数百毫秒降低到几十毫秒。

## 技术架构

流量路径：

```
用户浏览器 → DNS 解析 → CNAME 链 → 优选 IP 调度器 → Cloudflare 边缘节点 → SaaS 路由 → Tunnel → 本地服务器
```

核心组件：

1. **IP 调度器** — 根据用户网络条件返回最优 CDN 节点
2. **Cloudflare for SaaS** — Custom Hostnames 解耦访问域名和隧道域名
3. **DNS CNAME 链** — 通过优选 IP 路由请求

## 配置步骤

### 前置要求

- 两个域名（主域名和辅助域名）
- Cloudflare 账号

### 配置流程

1. **在辅助域名配置 Cloudflare Tunnel**
2. **启用 SaaS Custom Hostnames**
3. **添加 DNS 记录**
   - TXT 记录用于 SSL 验证
   - CNAME 记录用于路由
4. **配置 fallback 源记录**
5. **等待 SSL 证书激活**

<Callout type="important">
  必须关闭橙色云朵（代理状态设为 "DNS only"），否则会中断 DNS 解析链！
</Callout>

## 验证配置

配置完成后，测试访问速度是否有改善。可以使用 ping 或 curl 测试延迟。

## 常见问题

### 配置后无法访问？

检查 DNS 记录是否正确，确保关闭了 Cloudflare 代理（橙色云朵）。

### SSL 证书未激活？

等待 Cloudflare 完成证书签发，通常需要几分钟到几小时。
